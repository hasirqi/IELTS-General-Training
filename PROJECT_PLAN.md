## 不可破坏的边界

- 不把内部结果称为 MetaMetrics 官方 Lexile 或蓝思认证。
- 不把接受性词汇量称为写作或口语产出能力。
- 不用词汇量直接换算 IELTS 总分。
- 不把官方 VST 与自研 CAT 做简单比例换算。
- 不复制无再发布许可的官方题目或频率数据到公开仓库。
- 不让未人工审核锚点进入正式计分。
- 没有群体数据时不伪造 2PL 区分度。
- AI 不决定正确答案，也不自由改变测量范围。
- 个人数据默认只保存在 IndexedDB。

## M1：20K 词族测量底座

### M1.1 定义词族规则

先定义并用测试样例固定：词族代表词、屈折变化、透明与不透明派生、多词词块、专有名词、数字日期、缩写、拼写变体和同形异义。

### M1.2 正式字段

每个词族至少记录：familyId、headword、members、frequencyRank、frequencyBand、partsOfSpeech、syllables、ageOfAcquisition、concreteness、derivationType、properNoun、numberLike、teachingLexiconIds、source、licence、reviewStatus 和 version。

### M1.3 来源与许可证

对每个来源记录名称、版本、下载地址、原作者、许可证、是否允许修改、是否允许公开再发布、是否只允许研究使用，以及项目实际保存原始内容还是派生统计。

### M1.4 映射 4,133 教学词

输出映射成功数、一词多族歧义、多词归并同族、专有名词、数字、透明派生、各频率层覆盖率和未匹配清单。

### M1.5 门禁

- 正好 20,000 个唯一 familyId。
- 频率排名连续。
- 每个词族只有一个主层级。
- 透明派生不重复膨胀。
- 来源与许可证不为空。
- 4,133 教学词都有明确处理结果。
- 自动测试覆盖归并、特殊词、重复和统计。

Git 节点：Build 20K word-family measurement index。

## M2：150 锚点复合词汇 CAT 垂直切片

状态：已完成并发布（提交 `76cf9f4`）。

### 锚点与题型

- 1K–5K 每层 24 个，6K–8K 每层 10 个，共 150 个。
- 每题包含词族、频率层、词性、聚焦中文义、真实英文语境、项目原创英文释义、三个同词性英文干扰项、三个中文干扰项、初始参数和审核记录。
- 正式词汇 CAT 使用“英文关键词＋英文短语境＋四选一英文释义”。
- 英中四选一降级为每日快速自测，不进入正式结果。

### 基础路由与可信度

- 固定 15 题：12 个覆盖 1K–5K 的真实词＋3 个非词。
- Yes/No 只确定 CAT 起点和可信度，不进入词汇量分数。
- 非词不计分、不进入错词；误认多个非词会降低结果可信度。

### 浏览器端引擎

1PL/EAP 正式运行，数据接口兼容未来 2PL。正式 CAT 最少 20 题、最多 30 题；达到标准误差和频段覆盖后停止，18 分钟强制停止。草稿与结果由 IndexedDB 保存，旧版未完成草稿不会混入新规则。

### 测试中心

采用三层结构：紧凑推荐卡、五种并列模式、最近结果摘要。快速定位、词汇量精测和每日词汇自测已可用；阅读能力测评和语境运用练习保持可见但禁用，只有 720 题和 110 篇文章通过门禁后才能开放。

结果只报告宽频率层、区间、每层表现和可信度，不显示伪精确词数、官方蓝思认证、IELTS 换算或无样本支持的群体百分位。

Git 节点：Add 150-anchor contextual vocabulary CAT pilot。

## M3：600 锚点正式可用版

状态：进行中。600 的精确频段清单已经冻结；当前可计分 186，待审核候选 414；1K 已达到 60 条目标。不得把候选数写成已完成人工锚点数。

### 分布

- 1K–8K：每层 60，共 480。
- 9K–14K：每层 15，共 90。
- 15K–20K：每层 5，共 30。

### 内容顺序

按 1K–2K、3K–4K、5K、6K–8K、9K–14K、15K–20K 分批审核。

每批检查层级顺序、词族唯一、词性平衡、答案唯一、干扰项有效、中文自然度、跨批重复、伪词冲突、来源和许可证。

候选清单由 `scripts/build-vocabulary-anchor-candidates-600.mjs` 从 20K 词族索引、教学词映射和已审核教学语境确定性生成，产物为 `src/content/vocabulary-anchor-candidates-600.json`。生成器只接受一对一直接映射、唯一词族、目标词可在例句和搭配中核验的教学词，并按频段内目标分位点抽样，避免候选集中在频段起点。候选文件明确设置 `scoringEligible: false`，运行时 CAT 不得导入它。

### 正式输出

词族量点估计、置信区间、1K–20K 分层画像、可信度、与 4,133 教学词的交集和需要进入学习的高价值缺口。文案标记为“内部预标定接受性词族估计”。

### 重测

默认 4–6 周，保存近期题目，优先未见锚点；必要锚题可有限重复；过于频繁重测降低可信度。

完成节点：Release 600-anchor vocabulary assessment。

## M4：文本难度引擎

### V0 特征

实现文本清洗与句子识别、词族切分、平均句长及对数、平均词频对数、各频率层覆盖率、未登录词、音节和长词、从句和连接词、代词指代、名词化、抽象度、信息密度及词汇句法多样性。

### 六个内部层

- L1：基础识别和短句。
- L2：生活功能文本。
- L3：多段生活与工作文本。
- L4：复杂信息与一般推断。
- L5：抽象观点和较密集论证。
- L6：高级连续文本。

### 标定语料

目标至少 600 篇合法参考文本，每层约 100 篇，覆盖生活、工作、教育、公共服务、说明和论证。保存人工标签、来源和许可；不能再发布的文本只保存允许的特征、链接和哈希。训练、验证和最终保留测试集必须分离。

### 模型比较

比较透明线性基线、Ridge/Elastic Net 和轻量 GBDT，记录 MAE/RMSE、分层混淆、极端误差、特征重要性和版本。复杂模型没有明显提升时保留透明模型。

### 现有课程回标

给 128 节阅读课程计算内部难度、词族覆盖、关键生词和句法挑战，并人工抽查每层代表文本。发现倒挂应修正模型，不能手工改一个数字掩盖问题。

Git 节点：Add auditable internal text difficulty engine。

## M5：六批 720 题与 110 篇文章

每个难度层一个批次，共 120 题：40 道英文语境词义题（英文语境＋英文释义）、20 道完形、10 篇功能文本共 30 题、8–9 篇连续文章共 30 题。英中题与 Yes/No 路由不占 720 道正式计分题。

六批完成后正好达到 720 题、60 篇功能文本和 50 篇连续文章。

主题覆盖生活服务、住房、交通、健康、工作、培训、安全、消费、投诉、公共服务、社区、环境、社会议题、科普和一般抽象论证。

每篇人工标注关键生词、必须理解词、可推断词、低价值低频词、搭配、长句、主干、指代链、连接关系、题目证据和干扰项错因。

每题必须有唯一 ID、四个唯一选项、唯一答案、明确证据、自然英文题干、中文证据解释，不跨批重复、不含占位符。文本难度和题目难度分开保存。

六个批次分别测试、构建、提交、推送、等待部署并核对线上内容。

## M6：阅读 CAT 与三条路径

路径 A 为生活服务和公共信息；路径 B 为工作、培训和社区事务；路径 C 为消费、健康、环境和社会议题。共同锚题用于比较，主体文章不重复。

选题规则：文章成组出现；先中间难度定位；根据答案上下探；每次至少 2 篇功能文本和 3 篇连续文章；事实、主旨、指代、推断和篇章关系保持最低覆盖；回避近期文章。

停止规则：最少 38 题、最多 50 题、最长 30 分钟、覆盖至少 3 个难度层；初期 SE ≤ 0.35 theta 才允许提前结束；未收敛时给宽区间和低置信度。

结果输出内部模拟阅读值、区间、功能性与连续阅读表现、事实主旨指代推断篇章表现、速度和可信度。

Git 节点：Release adaptive reading assessment。

## M7：三轴能力坐标与学习回流

结果页同时显示教学词掌握、接受性词族及区间、内部模拟阅读值及区间、GT Reading 准备度、置信度和参数版本。

诊断至少支持：词汇低阅读低、词汇高阅读低、词汇一般阅读较高、阅读高而 IELTS 题型低、多个结果矛盾。

错误回流：生词进入复习；词义混淆进入词义实验室；长句误解进入句型实验室；主旨指代推断进入阅读专项；时间不足进入速度训练；IELTS 证据定位错误进入题型训练。

阅读护照显示内部文本难度、用户能力、已知覆盖、舒适/训练/冲刺、关键生词和篇章挑战。

Git 节点：Connect vocabulary reading and IELTS ability map。

## M9：外部区间校验与 IELTS 研究

用户可以手动录入官方 VST 结果，只比较区间是否重叠，不自动修改自研参数。允许本地录入 GT Reading 分数、考试日期和内部测试间隔，只做个人纵向观察。

群体研究只有另行授权才启动，届时需要自愿参加、隐私和删除机制、后端访问控制、样本计划、共同锚题、Rasch/2PL、DIF、外部效标和交叉验证。样本不足时不发布精确 IELTS 换算。

## 全程测试体系

### 数据测试

20K 词族唯一、频率层连续、透明派生处理、专有名词数字处理、锚点分布、状态门禁和许可证字段。

### 内容测试

答案与干扰项唯一、文章题干不重复、证据存在、无占位符、词性匹配、中文自然、主题和层级覆盖。

### 算法测试

单调性、随机猜测、全对全错、边界能力、高层偶然猜对、标准误差收敛、路径等价、重测回避、曝光控制和参数版本兼容。

### 产品测试

IndexedDB 保存迁移、中断恢复、离线、键盘、触控、屏幕阅读器、字号、iPhone、iPad、Windows、macOS 和 PWA 更新。

### 发布测试

每个节点依次执行专属测试、全量 pnpm test、pnpm build、git diff --check、明确暂存、提交推送 main、等待 GitHub Actions、核对线上版本和资源。

## 参数和结果版本

每次结果保存 engineVersion、wordFamilyIndexVersion、itemBankVersion、calibrationVersion、routeId、开始结束时间、theta、standardError 和 confidence。

模型更新不静默改写旧结果。保留原始答案；如用新模型重算，必须明确标记“重算”。

## 主要风险

- 数据版权：先审计再入库；个人非商业使用不等于可公开再发布。
- 假精度：参数状态明确区分词频初始、专家审核、实验和稳定标定。
- 内容量过大：按 150、600、1,000 锚点和六个文章层分批。
- 单用户无法群体标定：先用 1PL 和宽区间，2PL 群体标定另行授权。
- 测试疲劳：词汇和阅读拆分，允许保存草稿。
- 题目记忆：扩大题池、三条路径、4–6 周回避和曝光控制。
- 词汇量冒充英语能力：三轴分开并将错误回流到真正薄弱模块。
