# 破壁 IELTS 6：项目介绍与共同决策

> 这是项目的长期背景文档。无论在办公室、家里或新终端继续，都先阅读本文件，再阅读 [详细项目计划书](PROJECT_PLAN.md) 和 [执行说明](AGENTS.md)。

## 一分钟了解项目

“破壁 IELTS 6”是一套面向英语词汇和句型基础薄弱成年学习者的 IELTS General Training 本地优先学习 PWA。

它不是单纯背词网站，也不靠打卡、排行榜或幼稚激励维持使用。它要解决的是：单词看过却记不住、认识词形却不会用、逐词翻译读不懂句子、词汇增加却没有带动听说读写，以及不知道真正限制 IELTS 6 的究竟是词汇、句法、推断、速度还是题型。

长期目标：

- 6–9 个月持续训练。
- 每天约 30–45 分钟。
- IELTS General Training 总分达到 6.0。
- 内部参考毕业线为各单项不低于 5.5，并连续两次模拟达到目标。

口号是“英语难不倒我”，但口号保持低调，实用学习入口永远优先。

## 用户背景与教学语言

主要用户英语基础薄弱，懂中文、蒙古文和日文。

- 中文负责把概念讲明白。
- 日文和蒙古文只在关联准确且确有帮助时出现。
- 不在同一页面堆叠三种解释。
- 注意日语片假名发音偏差、假朋友和语义范围差异。
- 语气耐心、直接、不羞辱，不把成年学习者当儿童。

“我记不住”应被拆成可以训练的原因：词形不牢、意义片面、声音与拼写未连接、语境不足、句法误解、提取失败或复习时机错误。

## 已确定的产品体验

### 成人实用主义首页

首页直接提供听力、阅读、写作、口语四个入口，并保留复习旧词、新词与词块、单词学习、词汇量测试、句型实验室和开口任务。

不恢复大型“今天学什么”“今天完成几步”仪式，不用虚假打卡、排行榜或社交压力。

### 固定全站导航

顶部导航在全部页面保持一致并固定显示品牌、词库入口和“向 ChatGPT 提问”。课程区不重复添加相同入口。

### 本地优先

技术栈为 React、TypeScript、Vite 和可安装 PWA，目标覆盖 Windows、macOS、Chrome、Edge、Safari、iPhone、iPad、键盘、触控和屏幕阅读器。

学习记录当前保存在浏览器 IndexedDB。项目代码可以通过 GitHub 跨电脑同步，但个人学习进度不会随 Git 或浏览器自动同步。

### 不依赖 OpenAI API Key

核心学习不需要 API Key。ChatGPT 协作是手动闭环：产品复制结构化提示词并打开 ChatGPT 网页，用户获得反馈后粘贴回产品，本地保存。产品不能声称读取网页会话或自动获得回答。

## 词汇学习的长期原则

### 不宣称存在“雅思全部单词表”

正式教学词库由高频基础词、General Training 生活与工作场景词、固定搭配、用户错误词和真实阅读听力缺口组成。

是否从 4,133 扩到 6,000，不由“数字越大越好”决定，而由测评中反复出现的高价值覆盖缺口决定。

### 分开训练形式、意义和使用

“看着眼熟”不算掌握。系统分别训练识别与拼写、核心义与常用义、搭配与句中使用。复习交替使用主动回忆、听音拼写、语境选义、填空、延迟回忆和口头使用。

中文释义遵循“核心义优先＋高频常用义补充”。高频多义词显示真正影响理解或表达的 2–4 个常用义，不堆冷僻词典义。“认识”只表示词形和词义熟悉，不能直接计为使用掌握。

### 复习来自真实学习状态

复习优先未解决错误、到期内容、遗忘风险高、反应慢和提示多的项目。错误必须重新出现，不能用固定样本冒充复习系统。

## 四项课程边界

- 听力正式课程使用仓库内固定英式 MP3；学习模式可回放，模考模式只能播放一次、禁止拖动，提交后显示原文。
- 阅读使用 General Training Section 1、2、3，题目答案必须有原文证据，错因区分词汇、句法、指代、主旨、推断、证据定位和时间不足。
- 写作覆盖 GT Task 1 正式、半正式、私人书信和 Task 2 议论文，不混入 Academic Task 1 图表题。
- 口语覆盖 Part 1、2、3，必须真实开口；示范用于理解结构，不用于机械背诵。
- AI 不能修改客观题答案，也不能自由决定课程范围。

## 为什么下一阶段不是继续堆词

4,133 条教学词已经达到“先扩到 4,000，再评估是否继续到 6,000”的节点。下一步要回答三个不同问题：

1. 用户实际认识多少接受性词族？
2. 用户能读懂多难的真实文章？
3. 用户对 IELTS General Training Reading 的准备度如何？

三者相关，但不能互相冒充。词汇高而阅读低，可能是句法或推断问题；阅读高而 IELTS 得分低，可能是题型、限时和证据定位问题；认识大量词也不代表能在写作和口语中准确产出。

## 双方脑洞如何融合

用户提出的关键构想：

- 建立类蓝思文本难度引擎，从平均句长和平均词频对数起步，再加入音节、稀有度、抽象度、句间复杂度和文本密度。
- 使用 Rasch/IRT 把文本难度和读者能力放在可比较尺度。
- 先人工标定 500–1,000 个锚点词，再用 IRT/CAT 自适应测试。
- 使用少量伪词和词义确认题过滤“自称认识但实际不认识”。
- 建设人工审核的分级文章库，并人工标注关键生词，为阅读护照服务。
- 用真实 IELTS GT Reading 成绩做后续外部映射，不照搬网络换算表。
- Python 研究工具负责参数拟合，浏览器引擎负责正式运行。

在此基础上补充并修正：

- 教学词库、测量词族索引、阅读能力和 IELTS 准备度必须分开。
- 文本难度不等于题目难度；文章、问题和读者分别建模。
- 20K 是后台测量上限，不是要求用户学习 20,000 词。
- 先做 150 锚点验证，再做 600 可用、1,000 完整，避免等全部内容完成才验证引擎。
- 未审核锚点不能因为“边用边补”进入正式计分。
- 没有群体作答数据时不能真实估计 2PL 区分度，先用 a=1 的 1PL/Rasch 稳定运行。
- 伪词只用于有效性检查，不是正式词汇题主体。
- 官方 VST 与自研 CAT 只做区间重叠校验，不做比例换算。
- 内部模拟 L 值必须由实际阅读表现决定，词汇测试只负责起始定位和诊断。
- 当前产品是个人、本地优先系统；群体 Rasch 和 IELTS 映射属于未来自愿研究阶段。

最终融合架构：

词族测量底座 → 锚点与词汇 CAT → 文本难度引擎 → 人工文章与阅读 CAT → 三轴能力坐标 → 学习回流 → 可选的真实 IELTS 校验。

## 三轴能力坐标

### 词汇储备

同时显示：

- 正式教学词掌握，例如 2,860 / 4,133。
- 接受性词族估计，例如 4,700（4,100–5,300）。

前者来自真实学习状态，后者来自独立测评。

### 内部模拟阅读值

例如内部模拟 L 值 820L、区间 760L–900L、置信度“中”。该值主要由实际阅读理解决定，词汇测试只负责起始定位和诊断。

界面必须长期显示“内部模拟值，非 MetaMetrics 官方认证”。

### IELTS GT 准备度

例如 GT Reading 估分 5.0–5.5，并指出 Section 3 推断、长句理解或限时正确率等限制。总分不能由阅读单项直接推算，听说读写继续分别估计。

核心原则：词汇量决定大概从哪里开始，实际阅读表现决定内部模拟阅读值，IELTS 题型表现决定考试准备度。

测试题型分工已经固定：Yes/No 加非词用于基础路由与可信度；英文语境加英文释义用于接受性词汇 CAT；英中选择用于日常教学自测；完形用于功能文本语境运用；长文主旨、细节、指代和推断用于连续阅读 CAT，并最终决定内部模拟 L 值。五类结果不能互相冒充。

## 20K 词族测量平台

### 教学词库与测量索引分开

- 教学词库：当前 4,133 条，可学习、复习并进入错词档案。
- 测量索引：最终覆盖 20,000 个英语词族，只用于词频分层、CAT 选题、文本覆盖、难度计算和上限探测。

20K 不会全部展示给用户背诵。选择 20K 是因为 10K 对长期使用可能过早封顶，而 30K、40K 会大量进入生僻文学词、历史词和专业术语。1K–14K 是重点精度区，14K–20K 只做高阶上限探测。

## 三阶段锚点建设

### 150 锚点验证版

1K–5K 每层 24 个，共 120；6K–8K 每层 10 个，共 30。用于跑通路由、CAT、保存、恢复、停止规则和路径一致性，只报告宽泛能力层，不输出伪精确词数。

### 600 锚点可用版

1K–8K 每层 60，共 480；9K–14K 每层 15，共 90；15K–20K 每层 5，共 30。开始输出点估计、置信区间、分层画像和可信度。

### 1,000 锚点完整版

补齐 1K–14K 精度和 14K–20K 上限，支持长期重测、更多路径和更严格曝光控制。

## 低阶分层＋高阶 CAT

流程为：

1. 用 12–18 题检查 1K–5K 基础层。
2. 只用它确定 CAT 起点，不据此给最终词汇量。
3. 在相邻难度层选择信息量最大的题。
4. 必要时向 8K、14K、20K 上探。
5. 输出点估计、区间、层级画像和可信度。

词汇 CAT 预计 30–45 题、12–18 分钟。阅读 CAT 单独使用 38–50 题、20–30 分钟。两者可分两次完成并合并报告，不能用十几道词汇题同时冒充完整词汇量和阅读能力。

## 自主研发测量引擎

浏览器 TypeScript 运行引擎负责答题概率、EAP/MAP 能力更新、题目信息量、自适应选题、标准误差、频率层约束、锚题与曝光控制、4–6 周近期题目回避、异常作答和 IndexedDB 恢复。

接口从第一天兼容 2PL，但没有群体数据时所有新区分度使用 a=1，实际按 Rasch/1PL 稳定运行，不能凭感觉伪造区分度。

Python 离线工具未来负责 1PL/2PL 拟合、CAT 模拟、题目拟合、局部依赖、DIF、路径等价和参数导出。catsim、girth、py-irt 或 R mirt 可用于研究和交叉验证，但生产运行不依赖 Python 服务或第三方 API。

单个用户的重复答题无法可靠估计每道题的区分度。没有自愿群体数据之前，系统保持“专家预标定／内部模拟”。

## 锚点内容门禁

锚点状态依次为：frequency-seeded → expert-reviewed → item-authored → pilot-active → calibrated。

- 词频初始分层只能用于候选分析。
- 完成词族、词义、词性和语境审核后才算专家审核。
- 完成题干、正确答案和三个有效干扰项后才算题目完成。
- 实验题可进入试验路径。
- 有足够证据后才进入稳定标定池。

未人工审核内容不能进入正式计分。

## 陷阱词、确认题和可信度

- 快速“认识／不认识”只负责寻找起始频段；其中的非词用于检查虚报风险，路由题本身不进入正式分数。
- 正式 CAT 以英文关键词＋英文语境＋英文释义四选一为主体；英中四选一只用于不计分的每日快测。
- 每次最多少量有效性检查词，伪词比例不超过约 5%。
- 伪词不计入词汇量、不进入错词本和学习词库，只影响本次可信度。
- 可信度还检查确认题正确率、反应时间、异常连续点击、层级倒挂、词族矛盾和锚题稳定性。

## 文本难度引擎

不能声称复刻官方 Lexile。正式名称为“内部文本难度／内部模拟 L 值”。

V0 先实现可解释特征：词数、句数、平均句长和对数、平均词频对数、各词族层覆盖率、未登录词、音节和长词、从句、连接词、代词与指代、名词化、抽象度、信息密度及词汇句法多样性。

V1 使用合法标定语料比较线性回归、Ridge/Elastic Net 和轻量 GBDT。如果复杂模型没有显著提升，就保留更透明模型。

低于约 650L 的早期阅读增强属于 V2。项目面向成年 IELTS 学习者，先把中高难度做稳，界面不能变成儿童产品。

## 720 道题和人工文章库

| 类型 | 数量 |
|---|---:|
| 语境词义 | 240 |
| 句中选词／完形 | 120 |
| 功能性短文阅读 | 180 |
| 连续文章阅读 | 180 |
| 合计 | 720 |

同时建设 60 篇功能性短文、50 篇连续阅读文章、六个难度层和三套以上测评路径。

每层同步完成 40 道语境词义、20 道完形、10 篇功能文本共 30 题、8–9 篇连续文章共 30 题，总计 120 题。

文章不能只按公式生成，必须人工标注主题、体裁、关键生词、必须理解词、可推断词、低价值词、搭配、长句主干、指代链、连接关系、题目证据和每个干扰项错因。这些数据同时生成“阅读护照”。

## 文本、题目和读者必须分开

同一文章可以出很容易的事实题，也可以出很难的跨段推断题。数据层分别保存文本难度、题目难度、区分度、猜测参数、题型、证据范围和标定版本。

题目初始难度由文本难度、题型、证据距离、推断层数、干扰项强度和时间压力共同决定。

## 阅读 CAT

- 38–50 题，20–30 分钟。
- 至少 2 篇功能文本和 3 篇连续文章。
- 至少覆盖 3 个难度层。
- 初期目标约为 SE ≤ 0.35 theta。
- 未收敛则继续到最大题量，并降低结果置信度。
- 文章成组出现，不能逐题随机切换。
- 没有完成足够真实阅读时不能给出模拟 L 值。

## 阅读护照与学习回流

每篇材料显示内部文本难度、用户能力、已知词族覆盖、舒适／训练／冲刺状态、关键生词和句法篇章挑战。

错误回流：关键词不认识进入词汇复习；认识单词但误解句子进入句型实验室；主旨和推断错误进入阅读专项；时间不足进入同难度速度训练；文章理解正确但 IELTS 题目错误进入题型与证据定位训练。

## IELTS 映射边界

第一阶段只研究 IELTS General Training Reading，不同时展开中考、高考、六级、考研和托福。

官方 VST 与自研 CAT 只做区间重叠校验，不能用简单比例反向修改参数。小样本只能展示观察区间，不能发布精确换算。总分 6.0 仍需结合听力、写作和口语。

真正的群体 Rasch/2PL 和 IELTS 映射需要多能力层真实用户、共同锚题、隐私同意、匿名化、删除机制和后端控制。当前产品只服务个人，默认不上传成绩与作答。

## 是否扩到 6,000 教学词

只有出现以下证据才继续：目标文本覆盖长期不足、5K–8K 未知词反复造成失败、模拟阅读连续停滞于词汇缺口或 GT 场景存在明确高价值空白。如果主要问题是句法、指代、推断、篇章或速度，则优先训练对应能力。

## 从任何终端继续

1. 检查仓库和本地修改。
2. 阅读本文件。
3. 阅读 PROJECT_PLAN.md。
4. 阅读 AGENTS.md 的检查点和执行纪律。
5. 确认最近 Git 提交。
6. 运行 pnpm install --frozen-lockfile、pnpm test 和 pnpm build。
7. 从计划书中第一个未完成里程碑继续。

禁止从旧压缩包、浏览器缓存或记忆中的旧数字继续开发。
